3. Universal Design Rules — All Channels

Template Layer Stack (EVERY post follows this)

LAYER 1: bg_image — Article's og:image. Fill mode. Center crop.
  FALLBACK: Solid primary color of the channel.

LAYER 2: gradient_overlay — Bottom 65% of canvas.
  linear-gradient(to top, [primary_color] 0%, [primary_color]CC 40%, transparent 100%)
  This makes ANY photo work with white text.

LAYER 3: category_tag (if applicable) — Top-left, 24px margin.
  Pill shape: accent color bg, 8px vertical / 14px horizontal padding.
  Text: DM Sans SemiBold, 10px, white, uppercase, 0.1em tracking.

LAYER 4: headline — Bottom third, 40px left/right margin.
  Channel-specific font (see above), white.
  Max 2 lines. If longer, reduce font size by 15%.

LAYER 5: summary (optional) — Below headline, 8px gap.
  DM Sans Regular, 15px, white at 70% opacity. Max 1 line.

LAYER 6: source_credit — Bottom-left, 24px margin.
  JetBrains Mono, 11px, white at 50% opacity.

LAYER 7: channel_logo — Bottom-right, 24px margin, 48×48px.
  Always white version. Never color on dark.

LAYER 8: accent_bar — Full width, 3px height, accent color.
  Position: bottom edge (Mandate, Rong) or top edge (Ledger, Dheu, Jonotar).

What NOT to Do (Mandatory for Designers)

  • NEVER use more than 2 fonts per post.
  • NEVER place text on a photo without gradient overlay. Readability is non-negotiable.
  • NEVER use colored text on colored backgrounds. White text on dark overlay ONLY.
  • NEVER stretch or distort images. Always "fill" crop.
  • NEVER use drop shadows on text. The gradient handles readability.
  • NEVER make the logo bigger than 48px on 1080×1080. It's a watermark, not a billboard.
  • NEVER use more than 1 accent color per post.
  • 40% whitespace minimum. Information feels curated, not crammed.
  • If you can't read the headline at phone-thumbnail size (200px width), it's too small.

What it is: An open-source real-time intelligence dashboard that aggregates 344 news sources, geopolitical signals, markets, military activity, and infrastructure data into one API. Free. Self-hostable. Covers exactly what your co-founder described: international events that impact Bangladesh.

Setup — Add to Phase 2 (Day 15+)

# On your GEX44 server, add World Monitor to docker-compose.yml:

  worldmonitor:
    image: node:20-alpine
    restart: unless-stopped
    working_dir: /app
    volumes:
      - ./worldmonitor:/app
    ports:
      - "5173:5173"
    command: sh -c "npm install && npm run dev"
    networks:
      - media-engine

# Then clone the repo:
cd ~/media-engine
git clone https://github.com/koala73/worldmonitor.git
docker compose up -d worldmonitor

# In n8n, create a new workflow:
# W06_WorldMonitor_Feed
# Cron: every 30 minutes
# HTTP Request → http://worldmonitor:5173/api/events?region=south-asia
# Filter: events mentioning Bangladesh, India-BD, tariff, IMF, World Bank
# Score → Route to Mandate/Ledger/Jonotar → normal pipeline
Why Phase 2, not Day 1? World Monitor is an additional intelligence layer. Your core pipeline (107 BD sources) must work first. Adding World Monitor on Day 1 = complexity before you've proven the basics. Add it when scaling — it becomes your competitive edge for international coverage.
